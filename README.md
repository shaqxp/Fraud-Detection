## Overview

This project detects fraudulent credit card transactions using machine learning. A trained model is served through an interactive Streamlit web interface where users can enter transaction details and instantly receive a fraud probability score along with a risk level classification.

---

## Features

- Interactive web UI built with Streamlit
- Real-time fraud probability prediction
- Risk level classification (Low / Medium / High)
- Configurable decision threshold (default: **0.40**)
- Pre-filled sample transaction values for quick testing
- Feature descriptions and interpretation guide

## Dataset

The model is trained on the [Credit Card Fraud Detection dataset](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud), which contains transactions made by European cardholders in September 2013.

- **284,807** transactions total
- **492** fraudulent transactions (~0.17% of all transactions)
- Features `V1`–`V28` are the result of a **PCA transformation** applied for privacy reasons
- `Time` and `Amount` are the only original (non-anonymized) features
- `Class` is the target label: `1` = Fraud, `0` = Legitimate

---

## Model

Two models were trained and compared in `LOG_RF.ipynb`:

| Model | ROC-AUC |
|---|---|
| Logistic Regression | reported in notebook |
| Random Forest | reported in notebook |

- Both models use `class_weight="balanced"` to handle the severe class imbalance.
- `Time` and `Amount` are scaled using `StandardScaler`; `V1`–`V28` are left as-is (already PCA-transformed).
- The final model is saved as `fraud_model.pkl` and the scaler as `scaler.pkl`.
- **Decision Threshold: 0.40** — a transaction is flagged as fraud when `predicted_probability >= 0.40`.

## Getting Started

### Prerequisites

- Python 3.8 or higher

### Installation

1. **Clone the repository**

   ```bash
   git clone https://github.com/shaqxp/Fraud-Detection.git
   cd Fraud-Detection
   ```

2. **Create and activate a virtual environment** (recommended)

   ```bash
   python -m venv venv
   source venv/bin/activate      # Linux/macOS
   venv\Scripts\activate         # Windows
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

### Running the App

```bash
streamlit run app.py
```

The application will open in your browser at `http://localhost:8501`.

> **Note:** `fraud_model.pkl` and `scaler.pkl` must be present in the same directory as `app.py`. These files are generated by running `LOG_RF.ipynb`. If they are missing, retrain the model using the notebook.

## Usage

1. Open the app in your browser.
2. Enter transaction values in the form:
   - **Time** – seconds elapsed since the first transaction in the dataset.
   - **Amount** – transaction amount.
   - **V1–V28** – anonymized PCA-transformed features.
3. Click **Check Transaction**.
4. View the **fraud probability**, **prediction** (Fraud / Safe), and **risk level**.

Default values pre-fill a sample legitimate transaction so you can test the app immediately.

## How Predictions Work

1. User inputs are collected and assembled into a single-row DataFrame.
2. `Time` and `Amount` are scaled using the saved `StandardScaler`.
3. The scaled input is passed to the trained model's `predict_proba` method.
4. The fraud probability (class `1` probability) is compared against the threshold:
   - **probability ≥ 0.40** → 🚨 **Fraudulent Transaction**
   - **probability < 0.40** → ✅ **Safe Transaction**
5. A risk level is assigned:
   - **< 0.30** → Low Risk
   - **0.30 – 0.69** → Medium Risk
   - **≥ 0.70** → High Risk
